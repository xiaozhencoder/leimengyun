# 雷檬云 — 糖尿病健康管理平台 产品需求文档（PRD）

> 版本：v1.0 | 阶段：P0 MVP | 更新日期：2026-02-28

---

## 一、产品概述

### 1.1 产品定位

面向 **糖尿病患者** 和 **医生** 的移动端健康管理平台，帮助患者记录血糖、饮食、用药数据，并通过即时通讯与医生建立持续的管理关系。

### 1.2 目标用户

| 角色 | 描述 | 核心诉求 |
|---|---|---|
| 患者 | 糖尿病患者（1型/2型） | 便捷记录健康数据、获得专业指导 |
| 医生 | 内分泌科医生、营养师等 | 远程管理患者、查看数据、提供指导 |
| 管理员 | 平台运营人员 | 用户管理、医生审核、数据统计 |

### 1.3 P0 范围

| 模块 | 包含 | 不包含（后续迭代） |
|---|---|---|
| 用户体系 | 注册/登录、患者档案、医生档案、医患绑定 | 第三方登录、实名认证对接 |
| 健康数据 | 血糖记录、饮食记录、用药记录、数据趋势图 | AI 碳水识别、CGM 设备对接 |
| 医患沟通 | 文字消息、图片消息、血糖卡片分享 | 语音消息、视频问诊、处方开具 |

## 二、用户体系

### 2.1 注册与登录

**登录方式：** 手机号 + 短信验证码

**流程：**

```
打开应用 → 输入手机号 → 获取验证码 → 输入验证码
→ 系统判断是否已注册
   ├─ 已注册 → 直接进入首页
   └─ 未注册 → 进入角色选择 → 填写基本信息 → 进入首页
```

**业务规则：**
- 验证码有效期 5 分钟，每日同一号码最多发送 10 次
- 首次登录自动注册，需选择角色（患者/医生）
- 角色选择后不可更改

### 2.2 患者档案

| 字段 | 必填 | 说明 |
|---|---|---|
| 昵称 | 是 | 2-20 个字符 |
| 头像 | 否 | 默认头像 |
| 性别 | 是 | 男/女 |
| 出生日期 | 是 | 用于计算年龄 |
| 糖尿病类型 | 是 | 1型/2型/妊娠期/其他 |
| 确诊日期 | 否 | 确诊糖尿病的日期 |
| 治疗方案 | 是 | 胰岛素泵(CSII)/多次注射(MDI)/口服药物/生活方式管理 |
| 身高(cm) | 否 | — |
| 体重(kg) | 否 | — |

### 2.3 医生档案

| 字段 | 必填 | 说明 |
|---|---|---|
| 真实姓名 | 是 | — |
| 头像 | 否 | 默认头像 |
| 所在医院 | 是 | — |
| 科室 | 是 | 如：内分泌科 |
| 职称 | 是 | 主任医师/副主任医师/主治医师/住院医师 |
| 执业编号 | 是 | 用于资质验证 |
| 擅长领域 | 否 | 文字描述，限 200 字 |
| 个人简介 | 否 | 文字描述，限 500 字 |

**审核流程：** 医生注册后状态为"待审核"，管理员审核通过后方可被患者搜索和绑定。

### 2.4 医患绑定

- 患者通过搜索医生姓名或扫描医生二维码发起绑定申请
- 医生确认后建立绑定关系
- 一个患者可绑定多个医生，一个医生可管理多个患者
- 绑定后自动创建 IM 会话

## 三、健康数据记录

### 3.1 血糖记录

**记录字段：**

| 字段 | 必填 | 说明 |
|---|---|---|
| 血糖值 | 是 | 数值，单位 mmol/L，范围 1.0-33.3 |
| 测量时段 | 是 | 空腹/早餐前/早餐后/午餐前/午餐后/晚餐前/晚餐后/睡前/凌晨/随机 |
| 记录时间 | 是 | 默认当前时间，可修改 |
| 备注 | 否 | 限 200 字 |

**血糖等级判定（mmol/L）：**

| 等级 | 空腹 | 餐后2h | 颜色标识 |
|---|---|---|---|
| 偏低 | < 3.9 | < 3.9 | 🔵 蓝色 |
| 正常 | 3.9 - 6.1 | 3.9 - 7.8 | 🟢 绿色 |
| 偏高 | 6.2 - 7.0 | 7.9 - 11.1 | 🟡 黄色 |
| 高 | > 7.0 | > 11.1 | 🔴 红色 |

**数据展示：**
- 首页显示今日血糖曲线图（折线图）
- 支持查看 7 天 / 30 天趋势
- 列表展示历史记录，按日期分组

### 3.2 饮食记录

**记录字段：**

| 字段 | 必填 | 说明 |
|---|---|---|
| 餐次 | 是 | 早餐/午餐/晚餐/加餐 |
| 食物条目 | 是 | 至少一项。每项包含：食物名称、数量、碳水(g) |
| 拍照 | 否 | 上传餐食照片 |
| 总碳水 | 自动 | 系统自动求和 |
| 记录时间 | 是 | 默认当前时间 |
| 备注 | 否 | 限 200 字 |

### 3.3 用药记录

**记录字段：**

| 字段 | 必填 | 说明 |
|---|---|---|
| 药物类型 | 是 | 胰岛素/口服药 |
| 药物名称 | 是 | 如：诺和锐、二甲双胍 |
| 剂量 | 是 | 数值 |
| 剂量单位 | 是 | 胰岛素: U(单位)；口服药: mg/片 |
| 注射部位 | 否 | 仅胰岛素：腹部/大腿/手臂/臀部 |
| 记录时间 | 是 | 默认当前时间 |
| 备注 | 否 | 限 200 字 |

### 3.4 数据可视化

**首页仪表盘：**
- 今日血糖曲线（折线图，横轴时间，纵轴血糖值）
- 正常范围标注（绿色带 3.9-10.0 mmol/L）
- 今日统计卡片：记录次数、平均血糖、最高/最低值

**历史趋势（记录页）：**
- 7 天 / 30 天血糖趋势折线图
- 按日期分组的记录列表
- 可按类型筛选（血糖/饮食/用药）

## 四、医患沟通（IM）

### 4.1 会话管理

- 医患绑定后自动创建一对一会话
- 会话列表显示：对方头像、昵称、最后一条消息、时间、未读数
- 按最后消息时间倒序排列

### 4.2 消息类型

| 类型 | 说明 |
|---|---|
| 文字消息 | 限 500 字 |
| 图片消息 | 支持拍照或从相册选择，最大 10MB |
| 血糖卡片 | 患者可在聊天中分享最近的血糖记录卡片，医生可直接查看数值 |

### 4.3 消息状态

- 已发送 / 已送达 / 已读
- 离线消息：用户上线后自动拉取
- 未读消息数：在会话列表和底部 Tab 上显示红点/数字

### 4.4 医生端特殊功能

- 在聊天界面可快速查看该患者的最近血糖数据
- 点击患者头像可进入患者详情页，查看完整健康档案和历史数据

## 六、全局规则

### 6.1 设计规范

- **主色调：** #1AAD6E（医疗绿）
- **辅助色：** #FF6B35（警告橙）、#FF4D4F（危险红）、#3B82F6（信息蓝）
- **背景色：** #F7F8FA
- **字号：** 标题 18px、正文 14px、辅助 12px
- **圆角：** 卡片 12px、按钮 8px、输入框 8px
- **间距：** 页面边距 16px、卡片间距 12px

### 6.2 交互规范

- 所有表单提交需 loading 状态防重复点击
- 操作成功/失败需 Toast 提示
- 列表为空时展示空状态插画和引导文案
- 下拉刷新 + 上拉加载更多（列表页）

### 6.3 错误处理

- 网络异常：全局 Toast "网络连接失败，请稍后重试"
- 登录过期：自动跳转登录页
- 表单校验：实时校验 + 提交时校验，错误信息显示在字段下方
