# 雷檬云 — 问卷管理 产品需求文档（PRD）

> 版本：v1.0 | 阶段：P2 | 更新日期：2026-03-13

## 一、功能概述

### 1.1 功能定位

在雷檬云现有「健康数据记录 + 医患沟通 + 糖友社区」体系基础上，新增**问卷管理**模块，使医生能够通过标准化问卷对绑定患者进行定期随访评估，形成「数据记录 → 问卷评估 → 医患沟通 → 方案调整」的闭环管理链路。

### 1.2 核心价值

| 用户角色 | 价值 |
|---------|------|
| 医生 | 标准化评估患者状态；批量管理患者随访；量化治疗效果；降低院外随访成本 |
| 患者 | 定期自我评估健康状况；获得更精准的医嘱指导；增强自我管理意识 |
| 平台 | 积累结构化医疗数据资产；提升医患互动频率和粘性；为后续 AI 健康建议提供训练数据 |

### 1.3 竞品对标

| 维度 | 糖糖圈 | 雷檬云问卷（规划） | 差异化 |
|------|--------|------------------|--------|
| 随访形式 | 医院端随访系统（院内使用） | H5 端问卷（随时随地） | 不依赖院内系统，医患双端均可操作 |
| 问卷来源 | 纵向队列项目专用 | 平台预置模板 + 医生自定义 | 灵活度更高，医生可定制问卷 |
| 数据联动 | 独立数据库 | 与血糖/饮食/用药数据打通 | 问卷结果可自动关联健康数据 |
| 评估维度 | 以临床随访为主 | 自我管理 + 生活质量 + 临床指标 | 多维度综合评估 |
| 触达方式 | 院内扫码入组 | IM 消息推送 + 消息中心提醒 | 利用已有医患沟通链路，触达率更高 |

### 1.4 P2 范围

| 包含 | 不包含（后续迭代） |
|------|------------------|
| 平台预置问卷模板（T1DM 评估相关） | 医生完全自定义问卷编辑器 |
| 医生向绑定患者发送问卷 | 群发问卷、问卷广场 |
| 患者填写并提交问卷 | 问卷定时自动发送（定期随访计划） |
| 医生查看问卷结果和统计 | AI 智能分析报告 |
| 问卷结果关联患者健康数据 | 导出 PDF/Excel 报告 |
| 基础评分计算 | 复杂量表自动评分和解读 |

## 三、问卷模板设计

### 3.1 模板体系

平台预置基于 T1DM 临床评估标准的问卷模板，后续迭代支持医生自定义。

| 模板名称 | 评估维度 | 题目数 | 适用场景 | 参考标准 |
|---------|---------|--------|---------|---------|
| 血糖自我管理评估 | 血糖监测行为 | 10 | 定期随访 | SDSCA（糖尿病自我管理行为量表） |
| 饮食管理评估 | 饮食控制行为 | 8 | 营养指导 | SDSCA 饮食维度 |
| 运动管理评估 | 运动锻炼行为 | 6 | 运动指导 | SDSCA 运动维度 |
| 用药依从性评估 | 用药规范程度 | 8 | 治疗方案调整 | SDSCA + Morisky 量表 |
| 生活质量评估 | 生理/心理/社会 | 15 | 综合评估 | DSQL（糖尿病生存质量特异性量表） |
| 低血糖风险评估 | 低血糖事件频率与应对 | 8 | 安全评估 | 临床低血糖评估指南 |
| 足部护理评估 | 足部检查与护理行为 | 6 | 并发症预防 | SDSCA 足部维度 |
| 心理状态评估 | 糖尿病相关心理困扰 | 10 | 心理支持 | PAID（糖尿病问题领域量表） |

### 3.2 题目类型

| 题目类型 | 说明 | 示例 |
|---------|------|------|
| 单选题 | 从多个选项中选择一个 | "过去一周，您进行血糖监测的天数？" → 0-7天 |
| 多选题 | 从多个选项中选择多个 | "以下哪些情况会影响您的血糖监测？" → 多选 |
| 评分题 | Likert 量表评分 | "您对目前血糖控制的满意度？" → 1-5分 |
| 数值题 | 输入数值 | "过去一周，您的空腹血糖平均值约为？" → 数值输入 |
| 文本题 | 自由文本输入 | "您在血糖管理中遇到的最大困难是什么？" |
| 日期题 | 选择日期 | "您最近一次糖化血红蛋白检查的日期？" |

## 四、数据模型

### 4.1 新增数据库模型

```
// ==================== 问卷模板 ====================
model QuestionnaireTemplate {
  id            String      @id @default(uuid())
  title         String                              // 问卷标题
  description   String?     @db.Text                // 问卷说明
  category      QuestionnaireCategory               // 问卷分类
  questions     Json                                // 题目JSON数组
  totalScore    Int?        @map("total_score")     // 满分（可选）
  estimatedTime Int         @default(5) @map("estimated_time") // 预计填写时间(分钟)
  isSystem      Boolean     @default(true) @map("is_system")   // 系统预置 vs 医生自建
  creatorId     String?     @map("creator_id")      // 自建问卷的创建者
  creator       User?       @relation("TemplateCreator", fields: [creatorId], references: [id])
  status        TemplateStatus @default(ACTIVE)
  sortOrder     Int         @default(0) @map("sort_order")
  createdAt     DateTime    @default(now()) @map("created_at")
  updatedAt     DateTime    @updatedAt @map("updated_at")
  assignments   QuestionnaireAssignment[]

  @@map("questionnaire_templates")
}

// ==================== 问卷下发记录 ====================
model QuestionnaireAssignment {
  id            String      @id @default(uuid())
  templateId    String      @map("template_id")
  template      QuestionnaireTemplate @relation(fields: [templateId], references: [id])
  doctorId      String      @map("doctor_id")       // 发送医生的 userId
  doctor        User        @relation("AssignmentDoctor", fields: [doctorId], references: [id])
  patientId     String      @map("patient_id")       // 接收患者的 userId
  patient       User        @relation("AssignmentPatient", fields: [patientId], references: [id])
  status        AssignmentStatus @default(PENDING)
  deadline      DateTime?                            // 截止时间
  message       String?     @db.VarChar(200)         // 医生附言
  response      QuestionnaireResponse?
  createdAt     DateTime    @default(now()) @map("created_at")
  updatedAt     DateTime    @updatedAt @map("updated_at")

  @@map("questionnaire_assignments")
}

// ==================== 问卷作答记录 ====================
model QuestionnaireResponse {
  id            String      @id @default(uuid())
  assignmentId  String      @unique @map("assignment_id")
  assignment    QuestionnaireAssignment @relation(fields: [assignmentId], references: [id])
```

## 七、业务规则

### 7.1 问卷发送规则

| 规则 | 说明 |
|------|------|
| 发送对象 | 仅可向已绑定（ACCEPTED 状态）的患者发送 |
| 批量发送 | 单次最多同时向 20 位患者发送同一问卷 |
| 重复限制 | 同一模板对同一患者，14天内不可重复发送 |
| 截止时间 | 必填，最短1天，最长30天 |
| 医生附言 | 选填，限200字 |
| 取消规则 | 仅"待填写"状态的问卷可取消 |

### 7.2 问卷填写规则

| 规则 | 说明 |
|------|------|
| 填写方式 | 逐题模式，支持前后翻页 |
| 必填校验 | 标记为必填的题目未答时不可提交 |
| 暂存机制 | 自动保存到本地，下次打开恢复进度 |
| 提交限制 | 每份问卷仅可提交一次，提交后不可修改 |
| 过期处理 | 超过截止时间未提交，状态自动变为 EXPIRED |
| 填写时间 | 记录填写耗时，用于评估问卷质量 |

### 7.3 评分规则

| 规则 | 说明 |
|------|------|
| 自动评分 | 有分值的题目自动计算总分和维度得分 |
| 评分等级 | ≥80% 优秀 / 60-79% 良好 / 40-59% 一般 / <40% 较差 |
| 维度评分 | 按题目 dimension 字段分组计算 |
| 无分值题目 | 文本题、部分多选题不参与评分 |

### 7.4 通知规则

| 事件 | 通知对象 | 通知方式 |
|------|---------|---------|
| 新问卷发送 | 患者 | 消息中心通知卡片 |
| 问卷即将到期（到期前1天） | 患者 | 消息中心提醒 |
| 患者提交问卷 | 医生 | 消息中心通知 |
| 医生添加批注 | 患者 | 消息中心通知 |
| 问卷过期未填 | 医生 | 已发送列表状态变更 |

## 九、数据埋点

### 9.1 关键指标

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| 问卷发送量 | 每日/每周/每月发送问卷数 | 持续增长 |
| 问卷完成率 | 已完成 / (已完成 + 已过期) | ≥ 80% |
| 平均填写时长 | 所有已完成问卷的平均 duration | 合理范围内 |
| 医生批注率 | 已批注 / 已完成 | ≥ 50% |
| 评分改善率 | 同一患者同类问卷评分提升比例 | ≥ 60% |
